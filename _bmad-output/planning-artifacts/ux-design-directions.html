Key Design Decisions

Task Structure

How each task item is visually contained.

A,D Ruled lines — flat, notebook-like, tasks separated by borders
B Cards — each task elevated with rounded corners and shadow
C Minimal lines — same as ruled but with less visual weight

Checkbox Style

The affordance for completing a task.

A,D Circle — soft, friendly, coral or sage fill on completion
B Rounded square — modern, slightly more structural
C Square — minimal, stark, monochrome completion

Input Pattern

How the user adds a new task.

A,D Text input + coral "+" button — clear affordance for adding
B Inline input with text "Add" — softer, more integrated
C Bare input, Enter to submit — maximum minimalism, no button

Completed Tasks

Where completed tasks live in the list.

A,B,C Inline — completed tasks stay where they are, visually faded
D Separated — completed tasks move to a "Done" section below
